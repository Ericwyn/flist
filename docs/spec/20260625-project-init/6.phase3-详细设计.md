# Phase 3 详细设计：复制与收藏夹

> 对齐 `0.backend-design.md`、`1.开发阶段划分.md` 与 `5.存储驱动抽象设计.md`。承接 Phase 0 服务骨架、Phase 1 只读路径、Phase 2 写 / 搜索。本文聚焦 Phase 3「复制」「收藏夹」以及顺带落地的「跨平台磁盘用量 + 空间预检」。

## 1. 目标与边界

**目标**：实现文件 / 目录复制、跨目录移动（剪切粘贴）、目录收藏夹；前端以剪贴板交互（复制 / 剪切 / 粘贴）与侧边栏收藏夹承载。

**本阶段包含**：
- `POST /api/fs/copy`：复制（递归，磁盘空间预检，粘贴同名自动避让）
- 收藏夹 CRUD + 排序：`GET/POST/PUT/DELETE /api/bookmarks`、`PUT /api/bookmarks/reorder`
- `bookmarks` 表（存**相对 root 的路径**，root 迁移不失效、不越界）
- 跨平台磁盘用量封装（`disk_linux.go` / `disk_windows.go`）+ `storage.Usager` 接口（`local` + `Mux`），供复制空间预检使用，Phase 6 系统信息直接复用
- 复制 / 移动接口的 `auto_rename` 选项（粘贴同名自动避让）
- 前端：剪贴板状态（复制 / 剪切两态）、右键「复制 / 剪切 / 粘贴」、快捷键、粘贴进当前目录；侧边栏收藏夹（展示 / 跳转 / 失效标记 / 删除 / 拖拽排序）、「收藏当前目录」入口

**本阶段不包含**：上传（Phase 4）、打包下载与多选批量交互（Phase 5）、系统信息面板 UI（Phase 6，仅后端磁盘封装在此落地）。

**验收标准**：可跨目录复制；可跨目录移动（剪切粘贴）；同名粘贴自动避让不打断；收藏夹支持增删、拖拽排序、点击跳转；root 迁移后收藏仍有效，失效项有标记。

## 3. 关键设计取舍（已与用户确认）

| 取舍点 | 决策 | 说明 |
|--------|------|------|
| 磁盘空间预检 | **Phase 3 落地** | 顺带实现跨平台磁盘用量封装 + `Usager`，Phase 6 复用 |
| 同名冲突 | **粘贴自动避让** | 粘贴进目录同名时自动改名（`a.txt → a (2).txt`），不打断用户 |
| 自动避让作用范围 | **仅「移入已存在目录」分支** | 通过 `auto_rename` 选项控制；**重命名 / 移动到指定名仍冲突即报错**（走另一分支，保留 Phase 2 §5.3 语义） |
| 收藏对象 | **仅目录** | 符合「目录收藏夹快捷跳转」定位；创建时校验目标为已存在目录 |
| 失效检测 | **后端** | 列表返回时逐条 `Stat`，附带 `valid` 字段；NAS 收藏数量少，开销可忽略 |

> **为何 `auto_rename` 默认 false**：Phase 2 文档 §5.3 已明确「move 落点已存在 → 2004 file_exists」。为不与既有文档 / 测试冲突，接口默认保留严格语义；仅前端粘贴显式传 `auto_rename=true` 时启用避让。重命名（`dst` 为不存在的具体名）始终走严格分支，不受影响。

## 5. 磁盘用量封装（util/disk*.go）

```go
// util/disk.go
// Usage 返回 path 所在文件系统的总容量与可用字节数。跨平台 build tag 实现。
func Usage(path string) (total, free uint64, err error)
```

- **Linux**（`disk_linux.go`）：`unix.Statfs`，`total = Blocks * Bsize`，`free = Bavail * Bsize`（`Bavail` 为非特权用户可用块，比 `Bfree` 更贴近实际可写空间）
- **Windows**（`disk_windows.go`）：`windows.GetDiskFreeSpaceEx`，取 `lpFreeBytesAvailable`（调用方配额可用）与 `lpTotalNumberOfBytes`

`local.Local` 实现 `storage.Usager`：`Usage(ctx, p)` 经 `resolve(p)` 得本地路径后调 `util.Usage`。`Mux` 实现 `Usager`：`route` 到命中挂载点并委托其 `Usage`（挂载点后端未实现 `Usager` 时返回 `ErrNotSupported`）。

## 6. 复制接口（POST /api/fs/copy）

```json
// Request  { "src": ["/a/x.txt", "/a/sub"], "dst": "/backup", "auto_rename": true }
// Response { "code":0, "data": { "results": [ {"src":"/a/x.txt","ok":true}, ... ] } }
```

**dst 语义**（与 move 一致）：
- `dst` 是**已存在目录** → 每个 `src` 复制进该目录，落点 `dst/basename(src)`
- `dst` **不存在** → 仅 `len(src)==1` 合法，视为「复制到指定名」；多 src 每项 `not_a_dir`
- `dst` 是**已存在文件** → 每项 `not_a_dir`

**auto_rename 自动避让**（仅「移入已存在目录」分支生效）：
- 落点 `dst/basename` 已存在时，按 `name (2).ext` → `name (3).ext` … 递增探测首个不存在名（经 `backend.Stat`），上限 1000 次，超限则该项 `file_exists`
- 目录无扩展名时形如 `sub (2)`；文件保留扩展名 `a (2).txt`
- `auto_rename=false`（默认）：落点已存在 → 该项 `file_exists`（保留严格语义）
- 「复制到指定名」分支（dst 不存在）**不做避让**，落点已存在即 `file_exists`

**磁盘空间预检**（每项，仅当驱动支持 `Usager`）：
- 计算 `src` 总大小（文件取 `Stat.Size`；目录经 `Walk` 累加普通文件大小）
- 查询 `dst` 所在存储可用空间（`Usager.Usage`），不足 → 该项 `2005 disk_full`，不执行复制
- 驱动不支持 `Usager` 时跳过预检（无法判断，直接尝试复制）

**单项规则**（复用 `backend.Copy`）：落点 basename 校验、不复制进自身子树（`4000`）、src 不存在（`2001`）、越界（`2002`）。整体始终 `code:0` + 逐项 `results`（尽力而为）。

## 7. 移动接口的 auto_rename 扩展

`POST /api/fs/move` 请求体追加可选 `auto_rename`（默认 false）。语义与复制一致：仅「移入已存在目录」分支生效，落点同名时自动避让。前端「剪切 + 粘贴」传 `auto_rename=true`。**重命名**（`fs.rename` → dst 为不存在的具体名）走严格分支，行为不变。Phase 2 既有请求不带该字段，默认 false，语义与测试不受影响。

## 8. 收藏夹

### 8.1 数据表（store/db.go 追加，幂等）

```sql
CREATE TABLE IF NOT EXISTS bookmarks (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL,
    name        TEXT NOT NULL,
    path        TEXT NOT NULL,          -- 相对 root 的 API 路径（/ 分隔）
    sort_order  INTEGER DEFAULT 0,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
CREATE INDEX IF NOT EXISTS idx_bookmarks_user ON bookmarks(user_id);
CREATE UNIQUE INDEX IF NOT EXISTS idx_bookmarks_user_path ON bookmarks(user_id, path);
```

> 唯一索引 `(user_id, path)` 在库层兜底「同一路径不允许重复收藏」。

### 8.2 模型（model/bookmark.go）

```go
type Bookmark struct {
    ID        int64     `json:"id"`
    Name      string    `json:"name"`
    Path      string    `json:"path"`
    SortOrder int       `json:"sort_order"`
    CreatedAt time.Time `json:"created_at"`
    Valid     bool      `json:"valid"` // 运行时计算：目标仍存在且为目录
}
```

`UserID` 不对外序列化（单用户场景，按会话用户隔离）。

### 8.3 数据访问（store/bookmark.go）

```go
func (s *Store) ListBookmarks(userID int64) ([]model.Bookmark, error)            // 按 sort_order, id 升序
func (s *Store) CreateBookmark(userID int64, name, path string, sortOrder int) (int64, error) // 唯一索引冲突 → ErrBookmarkExists
func (s *Store) UpdateBookmark(id, userID int64, name string, sortOrder int) error             // 仅更新归属本人的记录
func (s *Store) DeleteBookmark(id, userID int64) error
func (s *Store) ReorderBookmarks(userID int64, orders []BookmarkOrder) error      // 事务批量更新 sort_order
func (s *Store) MaxBookmarkSort(userID int64) (int, error)                        // 新增时追加到末尾
```

`ErrBookmarkExists` / 复用 `ErrNotFound`。归属校验：所有按 `id` 的操作均带 `AND user_id = ?`，影响行数 0 视为 `ErrNotFound`，防止越权操作他人收藏。

### 8.4 服务层（service/bookmark.go）

`BookmarkService` 持有 `*store.Store` 与 `storage.Backend`（校验路径 + 失效检测）。

```go
func (s *BookmarkService) List(ctx, userID) ([]model.Bookmark, error)        // 逐条 Stat 填 Valid
func (s *BookmarkService) Create(ctx, userID, name, path string) (*model.Bookmark, error)
func (s *BookmarkService) Update(ctx, userID, id, name string) error          // 重命名（排序经 reorder）
func (s *BookmarkService) Delete(ctx, userID, id) error
func (s *BookmarkService) Reorder(ctx, userID, []order) error
```

**Create 校验**：
- `path` 经 `util.CleanAPIPath` 归一化
- `backend.Stat(path)` 必须成功且 `Type==dir`，否则 `ErrNotDir`（非目录 / 不存在不允许收藏）
- `name` 为空时回落为路径 basename（根则用「我的文件」）；长度上限 255
- 唯一性由 store 唯一索引兜底 → `ErrBookmarkExists`
- 新建 `sort_order = MaxBookmarkSort + 1`（追加末尾）

**List 失效检测**：对每条 `backend.Stat(path)`，成功且为目录则 `Valid=true`，否则 `false`（root 迁移 / 目录被删后标记失效，不报错）。

### 8.5 接口（handler/bookmark.go）

| 方法 | 路径 | 请求体 | 说明 |
|------|------|--------|------|
| GET | `/api/bookmarks` | — | 列表（含 `valid`） |
| POST | `/api/bookmarks` | `{name?, path}` | 添加，返回新记录 |
| PUT | `/api/bookmarks/{id}` | `{name}` | 重命名 |
| DELETE | `/api/bookmarks/{id}` | — | 删除 |
| PUT | `/api/bookmarks/reorder` | `{orders:[{id,sort_order}]}` | 批量排序 |

`userID` 取自 `middleware.UserFromContext`。`{id}` 经 chi URL 参数解析，非数字 → `4000`。

### 8.6 错误码（handler/errors.go 追加）

| Code | Message | 说明 |
|------|---------|------|
| 2005 | disk_full | 磁盘空间不足（复制预检） |
| 3001 | bookmark_exists | 同一路径已收藏 |
| 3002 | bookmark_not_found | 收藏不存在 / 非本人 |

## 11. 测试点

**后端**
- `util.Usage`（linux）：对 `t.TempDir()` 返回 `total>0 && free>0 && free<=total`
- `local.Usage` / `mux.Usage`：返回合理值；未知挂载点 / 不支持驱动错误归一
- `service.Copy`：复制文件（内容）、复制目录树、移入目录、复制到指定名、自身子树被拒、src 不存在；`auto_rename` 同名避让生成 `name (2)`；多 src dst 不存在 → not_a_dir
- 复制磁盘预检：构造超大 src 大小（mock / 跳过真实大文件，靠 size 计算逻辑单测）—— 以 `auto_rename` 与避让 helper 的单元测试为主，预检以「free 充足时正常复制」回归覆盖
- `service.Move` + `auto_rename`：移入目录同名避让；默认 false 时同名 `file_exists`（回归 Phase 2）
- `bookmark` store：创建 / 列表排序 / 重命名 / 删除 / 重复路径冲突 / 越权（他人 id）不可改删 / reorder 事务
- `bookmark` service：收藏目录成功、收藏文件被拒（not_a_dir）、收藏不存在路径被拒、失效检测（删目录后 valid=false）、name 缺省回落 basename
- `handler` e2e：copy→list 出现副本、同名 auto_rename 副本名、bookmarks 增删改查 + reorder、收藏文件 4000/not_a_dir、重复收藏 3001、跨用户隔离

**端到端冒烟**：登录 → 复制文件到子目录 → 同名再粘贴出现「(2)」→ 剪切粘贴跨目录 → 收藏当前目录 → 侧边栏跳转 → 拖拽排序 → 删除收藏。

## 12. 安全要点

- 复制 / 移动落点经 `backend.Copy/Move` 内的 `SafeResolve` + 自身子树检测，不逃逸 root、不自包含
- 收藏夹存**相对 root 路径**：root 迁移不失效、天然不越界；`Create` 经 `CleanAPIPath` + `Stat` 双重约束
- 收藏夹所有按 id 操作带 `user_id` 归属校验，防越权
- 复制属写操作，套写限流（10/s per IP）；收藏夹元数据操作受全局限流（50/s）
- 自动避让探测上限 1000 次，防御构造大量同名导致的探测放大

- 悬浮显示的取消收藏按钮（`StarOff`）维持原样。

### 13.2 搜索结果点击行为（FileBrowser.tsx `openHit`）
- 点击**文件**结果：直接打开预览框（`openPreview`），**停留在搜索结果页**（搜索范围本就是当前目录，跳转到所在目录无意义）。预览关闭后仍在结果列表。
- 点击**目录**结果：进入该目录并退出搜索（`clearSearch` + `navigate`）。
- `SearchResultsView` 的 `onOpen` 回调签名由 `(path, type)` 改为接收整个 `SearchHit`（预览需要 size/mode/modTime 构造临时 `FileEntry`）。

### 13.3 搜索态的整体约束（FileBrowser.tsx + fsStore.ts）
- **导航即退出搜索**：`fsStore.navigate` 在每次目录跳转开头统一清空搜索态，因此面包屑、收藏跳转、上级目录、目录结果点击等任何导航都会干净退出搜索，不残留半搜索状态。
- **搜索态顶栏形态切换**（§13.5）：搜索时左侧导航按钮组替换为「退出搜索」按钮，面包屑替换为结果标题。
- **操作区按钮在搜索态禁用**：新建文件夹 / 新建文件 / 属性 / 下载 / 显示隐藏 / 排序下拉 / 升降序 / 刷新 / 视图切换，均在 `searchOpen` 时灰显 `disabled`（这些操作针对当前目录，搜索结果页无意义）。
- 剪贴板快捷键（Ctrl/Cmd+C/X/V）与 F2 在 `searchOpen` 时不触发。

### 13.4 F2 重命名（FileBrowser.tsx）
- 选中单个条目时按 `F2` 打开重命名弹窗，贴近原生文件管理器。
- 搜索态、输入框 / 文本域聚焦、已有弹窗打开时不触发。

### 13.5 搜索结果页顶栏（FileBrowser.tsx）
- 进入搜索结果页时，顶部左侧整体切换：
  - 前进 / 后退 / 上级按钮组 → 单个「退出搜索」按钮（左箭头 + 文字），点击 `clearSearch` 返回目录浏览。
  - 面包屑路径 → 标题文字 `「{当前目录名}」目录下的搜索结果`（根目录显示「我的文件」）。
- 非搜索态完全保持原有的导航按钮 + 可点击面包屑。用 `searchOpen` 整体分支渲染。

### 13.6 搜索框增强（SearchBar.tsx + fsStore.ts）
- **递归开关**：搜索框内（清除 X 左侧）新增 `FolderTree` 图标按钮，切换是否递归子目录。
  - 状态：`fsStore.searchRecursive`，**默认 `false`（仅搜当前目录）**，由 `toggleSearchRecursive` 翻转。
  - 切换时若已有活动搜索，立即用新范围重搜。
  - **会话级**：刷新页面回到默认非递归，未持久化（有意为之，递归是临时意图）。
- **点击结果后清空输入框**：目录结果经 `searchOpen→false` 清空；文件结果监听 `searchOpen && previewPath` 清空（结果列表仍在预览框背后保留）。
- **最近搜索词下拉**：
  - 状态：`fsStore.searchHistory`，最多 10 条、去重置顶，**持久化到 `localStorage`（key `flist.searchHistory`）**，跨会话保留。每次 `runSearch` 成功记录。
  - 搜索框聚焦时下方弹出「最近搜索」下拉，点击即回填并立即搜索；有输入时按子串过滤。
  - 下拉右上角「清空」按钮调用 `clearSearchHistory`（含清 localStorage）；点组件外部 / Esc 收起。

### 13.7 工具栏按钮放大（FileBrowser.tsx）
- 顶部操作按钮点击区由 `p-1`/`p-1.5` 增至 `p-1.5`/`p-2`，图标由 `w-4 h-4`（16px）增至 `18px`，搜索框输入由 `text-xs py-1.5` 增至 `text-sm py-2`，整体更易点按。

### 13.8 明确**未做**项（留给后续 Phase）
- **多选交互（Shift/Ctrl 点选、全选）**：**本次刻意延后**，建议并入 Phase 5（批量操作 / 打包下载）。原因与现状：
  - 当前选中模型是 `fsStore.selected: string | null`（**单选**）。多选需重构为 `selectedNames: string[]` + range 锚点，牵涉 GridView/ListView 的点击与高亮、右键菜单、工具栏属性/下载按钮（仅单选可用）、复制/剪切快捷键、F2、状态栏计数等。
  - **底层已就绪**：`copyToClipboard` / `cutToClipboard` / `remove` 及后端 `fs.copy/move/remove` 均已接收数组，多选主要是前端「选中状态 + 交互」层的工作。
  - 用户期望：列表与图标视图都支持 Shift（区间）/ Ctrl（切换）多选。
- **递归搜索偏好持久化**：当前 `searchRecursive` 仅会话级；若要像主题那样长期记住，可挪到持久化 UI store。
- **磁盘用量面板 UI**：后端封装（§5）已在本阶段落地，前端状态栏展示属 Phase 6。
