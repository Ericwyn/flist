# flist 开发阶段划分

## 概述

将整体开发拆分为 7 个阶段，每个阶段完成后应具备可运行、可验证的完整功能。

---

## Phase 0：认证与服务骨架

**目标**：搭建可运行的服务端框架，完成用户认证体系。

- HTTP 服务器初始化（chi 路由 + 中间件管道）
- SQLite 数据库初始化（用户表、会话表）
- JWT 认证中间件 + 登录/登出接口
- PathGuard 路径安全中间件
- `/api/system/health` 健康检查接口
- 配置加载（命令行参数 + 环境变量）

**验收标准**：启动服务后可通过 `/api/auth/login` 获取 token，携带 token 访问受保护接口返回 401。

---

## Phase 1：文件浏览（只读）

**目标**：实现目录浏览和文件信息获取，前端可正常展示文件列表。

- `GET /api/fs/list` 目录列表（排序、隐藏文件控制）
- `GET /api/fs/stat` 单文件/目录详细信息
- `GET /api/fs/preview` 文件预览（文本内容、图片）
- 路径转换层（API 路径 <-> OS 路径）
- 前端文件浏览器基础 UI

**验收标准**：前端可浏览目录树，点击查看文件预览，跨平台路径正常工作。

---

## Phase 2：基础文件操作

**目标**：实现文件和目录的创建、修改、删除等写操作。

- `POST /api/fs/mkdir` 创建目录
- `POST /api/fs/touch` 创建空文件
- `POST /api/fs/move` 移动 / 重命名
- `DELETE /api/fs/delete` 删除文件 / 目录（递归）
- `GET /api/fs/search` 文件搜索
- 文件名校验（跨平台非法字符 + Windows 保留名）
- 前端右键菜单、重命名、删除确认弹窗

**验收标准**：可在前端完成新建、重命名、删除操作，搜索功能正常返回结果。

---

## Phase 3：复制与收藏夹

**目标**：实现文件复制功能和目录收藏夹。

- `POST /api/fs/copy` 复制文件 / 目录（递归）
- 收藏夹 CRUD 接口（增删改查 + 排序）
- SQLite 收藏夹表
- 前端复制粘贴交互（剪贴板状态管理）
- 前端侧边栏收藏夹展示与跳转

**验收标准**：可跨目录复制文件，收藏夹支持添加、删除、拖拽排序、点击跳转。

---

## Phase 4：文件上传与下载

**目标**：实现单文件的上传和下载。

- `POST /api/fs/upload` 单文件上传（multipart/form-data）
- `GET /api/fs/download` 单文件下载（Content-Disposition: attachment）
- 上传前磁盘空间检查
- 文件锁保护（并发写安全）
- 前端上传进度条、拖拽上传

**验收标准**：可上传任意文件到指定目录，可下载文件到本地，大文件上传不阻塞服务。

---

## Phase 5：批量操作与打包下载

**目标**：支持多文件同时操作和打包下载。

- 批量上传（多文件并行上传）
- 批量删除、批量移动
- 打包下载（服务端 zip 流式生成，选中多文件/文件夹后一键下载）
- 前端多选交互（Shift/Ctrl 点选、全选）

**验收标准**：可选中多个文件批量删除或下载，打包下载生成的 zip 内容完整。

---

## Phase 6：系统信息

**目标**：展示服务器端系统状态。

- `GET /api/system/info` 系统信息接口
- 磁盘空间检测（跨平台：Linux Statfs / Windows GetDiskFreeSpaceEx）
- 前端状态栏展示磁盘用量（总量、已用、剩余）

**验收标准**：前端底部状态栏正确显示磁盘空间信息，Linux / Windows 均可正常获取。

---

## 阶段依赖关系

```
Phase 0 ──> Phase 1 ──> Phase 2 ──> Phase 4 ──> Phase 5
                  \                    /
                   └──> Phase 3 ─────┘
                                      \
                                       └──> Phase 6
```

- Phase 0 是所有阶段的前置
- Phase 1 -> Phase 2 -> Phase 4 是主线
- Phase 3（收藏夹）可与 Phase 2 并行开发
- Phase 5 依赖 Phase 4 的上传下载基础
- Phase 6 可在任意阶段之后独立完成
