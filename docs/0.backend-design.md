# flist 后端技术设计文档

## 1. 项目概述

flist 是一个面向无界面 NAS 的 Web 文件管理器，提供文件在线浏览、查看、复制、粘贴、删除、移动、搜索等功能，并支持目录收藏夹快捷跳转。

技术栈：Go (后端) + Vite (前端) + SQLite (数据存储)

目标平台：Linux / Windows 双平台兼容

---

## 2. 跨平台兼容设计

### 2.1 核心原则

| 层面 | 策略 |
|------|------|
| API 路径格式 | 统一使用 `/` 作为分隔符，后端内部转换为 OS 本地路径 |
| 存储路径格式 | 数据库中存储的路径统一使用 `/` 分隔，读取时按运行平台转换 |
| 根目录表示 | Linux: `/mnt/data`，Windows: `D:/share` 或 `D:\share` |
| 路径处理 | 统一使用 `filepath` 包处理本地路径，`path` 包处理 API 路径 |

### 2.2 路径转换层

在 `internal/util/path.go` 中封装跨平台路径处理：

```go
// ToAPIPath 将 OS 本地路径转为 API 路径（统一用 /）
func ToAPIPath(osPath string) string {
    return filepath.ToSlash(osPath)
}

// ToOSPath 将 API 路径转为 OS 本地路径
func ToOSPath(apiPath string) string {
    return filepath.FromSlash(apiPath)
}

// JoinPath 安全拼接根目录与用户相对路径
func JoinPath(rootDir, relPath string) string {
    return filepath.Join(rootDir, filepath.FromSlash(relPath))
}
```

前端始终发送和接收 `/` 分隔的路径，后端在入口处转换，在出口处转回。

### 2.3 Windows 特殊处理

| 问题 | 处理方式 |
|------|----------|
| 盘符路径（`D:\`） | 配置的 `--root` 支持盘符格式，内部用 `filepath.Abs` 标准化 |
| 保留文件名（CON, PRN, NUL...） | 文件名校验增加 Windows 保留名检测 |
| 文件名非法字符 | Linux 仅禁 `/` 和 `\0`；Windows 额外禁 `<>:"/\|?*` |
| 路径长度限制 | Windows MAX_PATH=260，启用长路径时可达 32767；配置项控制是否启用 `\\?\` 前缀 |
| 隐藏文件判断 | Linux 看 `.` 前缀；Windows 需读取文件属性 `FILE_ATTRIBUTE_HIDDEN` |
| 符号链接 | Windows 需要管理员权限或开发者模式才能创建；列目录时仍解析 symlink 目标 |
| 文件锁 | Linux 用 `flock`；Windows 用 `LockFileEx` —— 统一用 `golang.org/x/sys` 封装 |
| 权限模式 | Windows 无 Unix 权限位，`os.FileMode` 仅部分有效；API 返回 mode 时 Windows 上返回简化表示 |

### 2.3.1 Windows 长路径处理策略

Windows 传统路径长度限制为 260 字符（MAX_PATH），但启用长路径支持后可达 32767 字符。为兼容两种情况，采用以下策略：

```go
// internal/util/path_windows.go

// EnableLongPath 启用长路径支持（需要 Windows 10 1607+ 且启用组策略）
func EnableLongPath() {
    // 通过注册表启用长路径：HKLM\SYSTEM\CurrentControlSet\Control\FileSystem LongPathsEnabled (DWORD) = 1
    // 或在应用清单中添加 <longPathAware>true</longPathAware>
}

// ToLongPath 将路径转换为长路径格式（添加 \\?\ 前缀）
func ToLongPath(path string) string {
    if len(path) >= 260 && !strings.HasPrefix(path, `\\?\`) {
        // 绝对路径且长度接近限制时添加前缀
        if filepath.IsAbs(path) {
            return `\\?\` + path
        }
    }
    return path
}

// SafeJoin 安全拼接路径，自动处理长路径
func SafeJoin(root, rel string) string {
    joined := filepath.Join(root, rel)
    return ToLongPath(joined)
}
```

**配置选项**：
```go
type Config struct {
    // ... 其他配置
    EnableLongPath bool   // 是否启用长路径支持（默认 false，仅 Windows 有效）
    MaxPathLength  int    // 路径长度警告阈值（默认 250）
}
```

**路径长度检查**：
```go
func CheckPathLength(path string, maxLen int) error {
    if runtime.GOOS == "windows" && len(path) > maxLen {
        return fmt.Errorf("path length %d exceeds limit %d", len(path), maxLen)
    }
    return nil
}
```

**注意事项**：
1. 长路径支持需要 Windows 10 版本 1607 或更高版本
2. 需要在组策略中启用"启用 Win32 长路径"或设置注册表
3. 使用 `\\?\` 前缀时，路径不能使用 `/` 作为分隔符，必须使用 `\`
4. 某些旧版应用程序可能不支持长路径，需要测试验证

### 2.4 文件名校验（跨平台）

```go
func ValidateFileName(name string, isWindows bool) error {
    if name == "" || name == "." || name == ".." {
        return ErrNameInvalid
    }
    if strings.ContainsAny(name, "/\x00") {
        return ErrNameInvalid
    }
    if isWindows {
        if strings.ContainsAny(name, `<>:"\|?*`) {
            return ErrNameInvalid
        }
        // 检查 Windows 保留名
        reserved := []string{"CON", "PRN", "AUX", "NUL",
            "COM1","COM2","COM3","COM4","COM5","COM6","COM7","COM8","COM9",
            "LPT1","LPT2","LPT3","LPT4","LPT5","LPT6","LPT7","LPT8","LPT9"}
        upper := strings.ToUpper(strings.TrimSuffix(name, filepath.Ext(name)))
        for _, r := range reserved {
            if upper == r {
                return ErrNameReserved
            }
        }
        // 文件名不能以空格或点结尾
        if strings.HasSuffix(name, " ") || strings.HasSuffix(name, ".") {
            return ErrNameInvalid
        }
    }
    return nil
}
```

### 2.8 跨平台文件锁实现

文件锁用于保护并发写操作，防止竞态条件。采用平台特定的实现：

```go
// internal/util/filelock.go

// FileLock 文件锁接口
type FileLock interface {
    Lock() error
    Unlock() error
    TryLock() error
}

// NewFileLock 创建文件锁
func NewFileLock(path string) FileLock {
    if runtime.GOOS == "windows" {
        return &windowsFileLock{path: path}
    }
    return &linuxFileLock{path: path}
}
```

**Linux 实现** (`internal/util/filelock_linux.go`)：
```go
//go:build linux

package util

import (
    "os"
    "syscall"
)

type linuxFileLock struct {
    path string
    fd   *os.File
}

func (l *linuxFileLock) Lock() error {
    fd, err := os.OpenFile(l.path, os.O_CREATE|os.O_RDWR, 0666)
    if err != nil {
        return err
    }
    l.fd = fd
    return syscall.Flock(int(fd.Fd()), syscall.LOCK_EX)
}

func (l *linuxFileLock) Unlock() error {
    if l.fd == nil {
        return nil
    }
    err := syscall.Flock(int(l.fd.Fd()), syscall.LOCK_UN)
    l.fd.Close()
    return err
}

func (l *linuxFileLock) TryLock() error {
    fd, err := os.OpenFile(l.path, os.O_CREATE|os.O_RDWR, 0666)
    if err != nil {
        return err
    }
    l.fd = fd
    return syscall.Flock(int(fd.Fd()), syscall.LOCK_EX|syscall.LOCK_NB)
}
```

**Windows 实现** (`internal/util/filelock_windows.go`)：
```go
//go:build windows

package util

import (
    "os"
    "syscall"
    "unsafe"
)

type windowsFileLock struct {
    path string
    fd   *os.File
}

func (w *windowsFileLock) Lock() error {
    fd, err := os.OpenFile(w.path, os.O_CREATE|os.O_RDWR, 0666)
    if err != nil {
        return err
    }
    w.fd = fd
    
    // 使用 LockFileEx 进行独占锁
    var overlapped syscall.Overlapped
    err = syscall.LockFileEx(
        syscall.Handle(fd.Fd()),
        syscall.LOCKFILE_EXCLUSIVE_LOCK,
        0,
        1, 0, // 锁定整个文件
        &overlapped,
    )
    return err
}

func (w *windowsFileLock) Unlock() error {
    if w.fd == nil {
        return nil
    }
    var overlapped syscall.Overlapped
    err := syscall.UnlockFileEx(
        syscall.Handle(w.fd.Fd()),
        0,
        1, 0, // 解锁整个文件
        &overlapped,
    )
    w.fd.Close()
    return err
}

func (w *windowsFileLock) TryLock() error {
    fd, err := os.OpenFile(w.path, os.O_CREATE|os.O_RDWR, 0666)
    if err != nil {
        return err
    }
    w.fd = fd
    
    var overlapped syscall.Overlapped
    err = syscall.LockFileEx(
        syscall.Handle(fd.Fd()),
        syscall.LOCKFILE_EXCLUSIVE_LOCK|syscall.LOCKFILE_FAIL_IMMEDIATELY,
        0,
        1, 0,
        &overlapped,
    )
    return err
}
```

**使用示例**：
```go
func SafeWriteFile(path string, data []byte) error {
    lock := NewFileLock(path + ".lock")
    if err := lock.Lock(); err != nil {
        return err
    }
    defer lock.Unlock()
    
    return os.WriteFile(path, data, 0644)
}
```

**注意事项**：
1. 文件锁是建议性的，不是强制性的
2. 锁文件应与目标文件在同一目录，避免跨文件系统问题
3. 程序崩溃时锁会自动释放（进程结束时）
4. Windows 上文件锁是强制性的，其他进程无法访问被锁的文件

### 2.9 SQLite 驱动选型调整

为避免 CGO 在 Windows 上的交叉编译困难，推荐使用纯 Go 实现：

| 驱动 | 说明 |
|------|------|
| `modernc.org/sqlite` | 纯 Go，无需 CGO，跨平台编译零障碍 |
| `github.com/mattn/go-sqlite3` | 需要 CGO + gcc，Windows 上需 MinGW |

**选择 `modernc.org/sqlite`** 作为 SQLite 驱动，确保 `go build` 在任何平台无需额外工具链。

### 2.10 文件权限在 Windows 上的表示

Windows 没有 Unix 风格的 `rwx` 权限位，API 返回 `mode` 字段需要做平台适配：

```go
// internal/util/perm.go

// FormatFileMode 跨平台格式化文件权限
func FormatFileMode(info os.FileInfo) string {
    if runtime.GOOS == "windows" {
        return formatWindowsMode(info)
    }
    return info.Mode().Perm().String() // 如 "0644"
}

func formatWindowsMode(info os.FileInfo) string {
    ptr, _ := syscall.UTF16PtrFromString(info.Name())
    attrs, err := syscall.GetFileAttributes(ptr)
    if err != nil {
        return "0644" // 默认值
    }

    var mode string
    if attrs&syscall.FILE_ATTRIBUTE_READONLY != 0 {
        mode = "0444" // 只读
    } else {
        mode = "0666" // 可读写
    }
    if info.IsDir() {
        mode = "0755" // 目录
    }
    return mode
}
```

API 返回的 `file_info` 结构中，`mode` 字段语义说明：

| 平台 | mode 含义 | 示例 |
|------|-----------|------|
| Linux | 标准 Unix 权限位 | `0755`, `0644` |
| Windows | 简化表示：只读=`0444`，读写=`0666`，目录=`0755` | `0666`, `0755` |

前端根据 `os` 字段（API 返回的系统类型）判断如何展示权限信息。

### 2.11 符号链接在 Windows 上的权限检查

Windows 上创建符号链接需要管理员权限或启用开发者模式。flist 不创建符号链接，但在列目录时需要正确处理已有的符号链接：

```go
// internal/util/symlink.go

// ResolveSymlink 跨平台解析符号链接
func ResolveSymlink(path string) (string, error) {
    info, err := os.Lstat(path)
    if err != nil {
        return "", err
    }

    // 非符号链接直接返回
    if info.Mode()&os.ModeSymlink == 0 {
        return path, nil
    }

    // 解析符号链接目标
    target, err := filepath.EvalSymlinks(path)
    if err != nil {
        if runtime.GOOS == "windows" {
            // Windows 上符号链接可能因权限问题无法解析
            // 记录警告但仍返回原始路径
            slog.Warn("symlink resolution failed, may need admin privileges",
                "path", path, "error", err)
            return path, nil
        }
        return "", err
    }

    return target, nil
}

// ListEntry 包含符号链接信息的目录条目
type ListEntry struct {
    Name      string `json:"name"`
    Type      string `json:"type"`       // "file", "dir", "symlink"
    IsSymlink bool   `json:"is_symlink"`
    LinkTarget string `json:"link_target,omitempty"` // 符号链接目标（仅前端展示用）
}
```

**列目录时的符号链接处理逻辑**：
1. 使用 `os.Lstat` 获取文件信息（不跟随符号链接）
2. 如果是符号链接，标记 `is_symlink: true`，尝试读取链接目标路径
3. 如果符号链接目标在根目录范围内，允许展示；否则标记为"目标不可达"
4. 不递归跟随符号链接，防止循环引用导致无限遍历

**Windows 注意事项**：
- 符号链接解析失败时降级处理，不阻塞目录列表
- 符号链接目标路径可能使用绝对路径，需在前端单独展示
- 交叉分区符号链接（如 C: -> D:）需额外校验目标是否在允许范围内

## 7. 逻辑设计

### 7.1 文件操作逻辑

#### 列表与浏览

- 所有路径以配置的根目录为基准，前端传入的 `path` 为相对路径，后端拼接 `rootDir + path` 得到实际路径
- 列表返回文件名、类型、大小、修改时间、权限模式等元信息
- 支持按名称、大小、修改时间排序
- 隐藏文件（以 `.` 开头）默认不显示，可通过参数控制

#### 复制与移动

- 支持批量操作（多个源路径）
- 复制目录时递归复制
- 移动操作优先使用 `os.Rename`（同分区），跨分区时退化为复制+删除
- 目标路径存在同名文件时返回冲突错误，由前端确认是否覆盖

#### 删除

- 支持批量删除
- 目录删除为递归删除（相当于 `rm -rf`）
- 删除前可选确认机制（前端负责二次确认，后端直接执行）

#### 搜索

- 基于文件名的模糊搜索（支持通配符或子串匹配）
- 支持递归与非递归两种模式
- 搜索设置超时时间（默认 30s），避免对超大目录树搜索时服务假死
- 使用 goroutine 并行遍历，通过 context 控制取消

#### 上传与下载

- 上传使用 `multipart/form-data`，支持大文件分片（前端切片后逐片上传）
- 下载直接以 `Content-Disposition: attachment` 响应
- 预览针对文本文件返回前 N 行内容，图片返回缩略图或原图

### 7.2 收藏夹逻辑

- 收藏夹存储的是绝对路径
- 浏览收藏时自动检测目标路径是否仍存在，不存在的标记为失效
- 支持手动排序（拖拽排序由前端发起 reorder 请求）
- 同一路径不允许重复收藏

### 7.3 认证逻辑

- 首次启动：若数据库无用户，使用配置的用户名密码创建管理员账户；若密码未配置则随机生成并打印到日志
- 登录：验证用户名密码，成功则签发 JWT（含 `sub`, `jti`, `exp`）
- 会话管理：每次签发的 Token 记录到 `sessions` 表，登出时标记 `revoked`
- Token 校验：中间件解析 JWT，检查是否过期、是否被吊销

## 8. 安全设计

### 8.1 路径穿越防护（核心）

这是整个系统最关键的安全点。所有文件操作必须确保路径不会逃逸出配置的根目录。

```go
func SafeResolve(rootDir, userPath string) (string, error) {
    // 1. 清理路径：去除 .., ., 多余斜杠
    cleaned := filepath.Clean(userPath)

    // 2. 拼接到根目录
    absolute := filepath.Join(rootDir, cleaned)

    // 3. 解析符号链接得到真实路径
    real, err := filepath.EvalSymlinks(absolute)
    if err != nil {
        return "", err
    }

    // 4. 确保解析后的真实路径仍在根目录内
    rootReal, _ := filepath.EvalSymlinks(rootDir)
    if !strings.HasPrefix(real, rootReal + string(filepath.Separator)) && real != rootReal {
        return "", ErrPathTraversal
    }

    return real, nil
}
```

关键要点：
- **必须在符号链接解析之后做前缀检查**，否则可通过 symlink 逃逸
- 所有 handler 入口处统一调用 `SafeResolve`，不允许在 service 层直接拼接路径
- PathGuard 中间件作为兜底层，拦截任何包含 `..` 或绝对路径的请求参数

### 8.2 认证安全

| 措施 | 说明 |
|------|------|
| 密码哈希 | 使用 bcrypt（cost=12），不存明文 |
| JWT 签名 | 使用 HMAC-SHA256，密钥长度 >= 32 字节 |
| Token 吊销 | 登出时标记 session revoked，中间件每次校验 |
| 登录限流 | 同一 IP 5 次失败后锁定 15 分钟 |
| 密码强度 | 修改密码时强制最少 8 位，含字母和数字 |

### 8.3 请求安全

| 措施 | 说明 |
|------|------|
| 全局限流 | 每 IP 每秒最多 50 请求 |
| 文件操作限流 | 写操作（上传/复制/删除）每秒最多 10 请求 |
| 请求体大小限制 | 非上传接口 body <= 1MB，上传接口按配置限制 |
| CORS | 仅允许同源或配置的来源 |
| Content-Type 校验 | 严格检查 Content-Type 是否与接口预期匹配 |

### 8.4 文件操作安全

| 措施 | 说明 |
|------|------|
| 禁止操作系统关键路径 | 黑名单拦截 `/proc`, `/sys`, `/dev` 等 |
| 符号链接限制 | 符号链接解析后目标必须仍在根目录内 |
| 文件名校验 | 禁止包含 `\0`、`/`（作为名称的一部分）等危险字符 |
| 递归操作保护 | 递归删除/复制设置最大深度（默认 100 层） |
| 并发写保护 | 同一文件的写操作使用文件级锁防止竞态 |
| 磁盘空间检测 | 上传/复制前检查剩余空间，不足时拒绝 |

### 8.5 传输安全

- 建议生产环境通过反向代理（Nginx/Caddy）配置 HTTPS
- 所有 API 响应添加安全 Header：
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: DENY`
  - `Content-Security-Policy: default-src 'self'`
  - `Strict-Transport-Security` (HTTPS 时)

### 8.6 日志与审计

- 所有写操作（创建、删除、移动、复制、上传）记录审计日志
- 日志内容：时间、操作类型、操作路径、来源 IP、操作结果
- 登录失败记录 IP 和尝试的用户名
- 日志输出到文件，支持按天轮转

```go
            t.Errorf("ValidateFileName(%q, false) should fail", name)
        }
    }

    // Windows 特有测试
    if runtime.GOOS == "windows" {
        winInvalid := []string{"CON", "PRN", "AUX", "NUL", "COM1", "LPT1",
            "test ", "test.", `a<b`, `a>b`, `a"b`, `a|b`}
        for _, name := range winInvalid {
            if err := ValidateFileName(name, true); err == nil {
                t.Errorf("ValidateFileName(%q, true) should fail on Windows", name)
            }
        }
    }
}
```

### 14.3 集成测试建议

- **CI 矩阵**：GitHub Actions 使用 `strategy.matrix` 同时在 `ubuntu-latest` 和 `windows-latest` 上运行测试
- **文件系统测试**：使用 `t.TempDir()` 创建临时目录，避免依赖外部文件系统
- **并发测试**：多 goroutine 同时操作同一文件，验证文件锁正确性
- **路径安全测试**：构造路径遍历攻击 payload，验证 `SafeResolve` 拦截
