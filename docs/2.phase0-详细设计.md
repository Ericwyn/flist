# Phase 0 详细设计：认证与服务骨架

> 对齐 `0.backend-design.md` 与 `1.开发阶段划分.md`。本文聚焦 Phase 0 的可落地细节（接口字段、表结构、中间件行为、目录结构、测试点），不涉及 Phase 1+ 的文件操作接口。

## 1. 目标与边界

**目标**：搭建可运行的服务端框架，完成用户认证体系与静态资源托管。

**本阶段包含**：
- HTTP 服务器（chi 路由 + 完整中间件管道）
- 配置加载（启动参数 + 环境变量，`--root` 必填）
- SQLite 初始化（`users` / `sessions` 表，WAL + `busy_timeout` + 单写连接）
- 不透明会话令牌认证 + 登录 / 登出 / 当前用户 / 改密
- `SafeResolve` 路径安全核心 + `PathGuard` 兜底中间件（为 Phase 1 打地基）
- 前端 `embed` 静态托管 + SPA history 路由 fallback
- `/api/system/health` 健康检查
- 前端登录页 + 鉴权门禁 + 登出

**本阶段不包含**（留待后续）：所有 `/api/fs/*` 文件操作、收藏夹、`/api/system/info` 磁盘信息、审计日志落库、登录失败状态持久化（Phase 0 用进程内锁定，见 §6.3）。

**验收标准**：服务启动后可通过 `/api/auth/login` 获取令牌；带令牌访问受保护接口成功、不带或失效令牌返回 401；前端静态页面可正常加载并刷新子路由不 404。

## 3. 配置设计

加载优先级：**启动参数 > 环境变量 > 默认值**。

| 配置项 | 环境变量 | 启动参数 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| 监听地址 | `FLIST_ADDR` | `--addr` | `:16550` | HTTP 监听地址（高位端口，避开常见端口冲突） |
| 根目录 | `FLIST_ROOT` | `--root` | **必填** | 缺失即启动失败 |
| 数据目录 | `FLIST_DATA` | `--data` | `./data` | SQLite 存放目录 |
| 管理员用户名 | `FLIST_ADMIN_USER` | `--admin-user` | `admin` | 初始管理员 |
| 管理员密码 | `FLIST_ADMIN_PASS` | `--admin-pass` | 随机生成并打印 | 初始管理员密码 |
| 会话有效期 | `FLIST_SESSION_TTL` | `--session-ttl` | `24h` | 会话过期时间 |
| 长路径支持 | `FLIST_LONG_PATH` | `--long-path` | `false` | 仅 Windows |

启动时对 `--root` 执行 `filepath.Abs` + `EvalSymlinks` 标准化并缓存为 `rootReal`，供 `SafeResolve` 使用；目录不存在则启动失败。

## 4. 数据库设计

打开参数：`?_pragma=journal_mode(WAL)&_pragma=busy_timeout(5000)&_pragma=foreign_keys(ON)`，写连接 `SetMaxOpenConns(1)`。

```sql
CREATE TABLE IF NOT EXISTS users (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    username    TEXT NOT NULL UNIQUE,
    password    TEXT NOT NULL,          -- bcrypt hash
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS sessions (
    id          TEXT PRIMARY KEY,       -- 会话令牌的 SHA-256 哈希（hex）
    user_id     INTEGER NOT NULL,
    expires_at  DATETIME NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_seen   DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
CREATE INDEX IF NOT EXISTS idx_sessions_user ON sessions(user_id);
CREATE INDEX IF NOT EXISTS idx_sessions_expires ON sessions(expires_at);
```

迁移用幂等 `CREATE TABLE IF NOT EXISTS`，Phase 0 无需版本化迁移框架。`updated_at` 由应用层显式赋值。

## 5. 认证设计

### 5.1 令牌

- 登录成功生成 32 字节随机值，Base64URL 编码为明文 token 返回客户端
- 服务端只存 `SHA-256(token)` 的 hex 作为 `sessions.id`，不存明文
- 校验：对收到的 token 取 SHA-256 后查表，命中且未过期则放行，并刷新 `last_seen`

### 5.2 首次启动建管理员

库中无用户时，用配置的用户名 / 密码创建管理员；密码未配置则随机生成 16 字节并以 `slog.Warn` 打印一次（仅此时打印）。

### 5.3 密码哈希

bcrypt（cost 10）。注意 bcrypt 72 字节截断：登录与改密前对密码做长度校验（> 72 字节拒绝）。改密强制 ≥8 位且同时含字母与数字。改密成功后吊销该用户的其他会话（保留当前会话）。

## 6. API 接口

统一响应：`{ "code": 0, "message": "success", "data": {} }`，错误时 `code` 非零。

### 6.1 认证模块

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/login` | 否 | 登录，返回令牌 |
| POST | `/api/auth/logout` | 是 | 登出，删除当前会话 |
| GET | `/api/auth/me` | 是 | 当前用户信息 |
| PUT | `/api/auth/password` | 是 | 改密（吊销其他会话） |

**POST /api/auth/login**
```json
// Request  { "username": "admin", "password": "xxx" }
// Response { "code": 0, "data": { "token": "<opaque>", "expires_at": 1700000000, "username": "admin" } }
```
令牌携带方式：`Authorization: Bearer <token>`（优先）或 Cookie `flist_session`。

**GET /api/auth/me** → `{ "code": 0, "data": { "id": 1, "username": "admin" } }`

**PUT /api/auth/password** → `{ "old_password": "...", "new_password": "..." }`

### 6.2 系统模块

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/system/health` | 否 | 健康检查 `{ "code":0, "data":{ "status":"ok" } }` |

### 6.3 登录限流（Phase 0 实现）

同 (IP, username) 维度，连续失败 5 次锁定 15 分钟。Phase 0 用**进程内**计数器实现（重启清零）；设计文档要求的「持久化、重启不丢」留到引入审计/限流持久化时补齐。锁定期间返回 `1003 account_locked`。

## 7. 中间件管道

```
Recovery → RequestID → Logger → RateLimit(global 50/s per IP) → CORS → SecurityHeaders
  → [public]   /api/system/health, /api/auth/login, 静态资源
  → [protected] Auth → PathGuard → /api/auth/{logout,me,password}
```

- **Recovery**：捕获 panic，记日志，返回 `9001`
- **RequestID**：生成请求 ID，写入响应头 `X-Request-Id` 与日志上下文
- **Logger**：`slog` 输出方法、路径、状态码、耗时、请求 ID、远端 IP
- **RateLimit**：每 IP 令牌桶（全局 50/s）；写操作 10/s 限流器构建好但 Phase 0 暂无写接口接入
- **CORS**：同源部署默认关闭；配置允许来源时按白名单放行（dev 调试用）
- **SecurityHeaders**：`X-Content-Type-Options: nosniff`、`X-Frame-Options: DENY`、基础 `Content-Security-Policy`
- **Auth**：取 token → 哈希 → 查会话 → 校验过期 → 注入用户上下文；失败 `1001 unauthorized`
- **PathGuard**：兜底拦截请求中 `path` 参数含 `..` 或绝对路径形态；Phase 0 无 fs 接口，作为管道占位与未来防线

## 8. 静态资源托管

- `web/embed.go` 用 `//go:embed all:dist` 嵌入前端产物
- 仓库提交真实的 `web/dist/` 构建产物（含 `index.html` 与 `assets/`），干净 checkout 无需先跑前端构建即可 `go build` 出可用二进制；前端改动后需重新构建并提交更新后的产物
- 处理器逻辑：请求路径命中真实文件 → 直接返回；以 `/api/` 开头 → 不接管（交给 404）；其余非文件路径 → 回退返回 `index.html`（SPA history 路由）

## 9. 错误码（Phase 0 用到的子集）

| Code | Message | 说明 |
|------|---------|------|
| 0 | success | 成功 |
| 1001 | unauthorized | 未登录或会话失效 |
| 1002 | invalid_credentials | 用户名或密码错误 |
| 1003 | account_locked | 登录失败次数过多 |
| 1004 | weak_password | 新密码不满足强度 |
| 2002 | path_traversal | 路径越界（PathGuard / SafeResolve） |
| 4000 | bad_request | 请求体非法 / 参数错误 |
| 9001 | internal_error | 服务端内部错误 |

## 10. SafeResolve 算法（Phase 0 实现，Phase 1 复用）

```
SafeResolve(rootReal, apiPath):
  1. cleaned = path.Clean("/" + trimLeft(apiPath, "/"))   // 消解 .. 与越根
  2. full    = filepath.Join(rootReal, fromSlash(cleaned))
  3. resolved, err = EvalSymlinks(full)
        若 not-exist：对父目录 EvalSymlinks 成功后拼 basename（支持创建类目标）
  4. 前缀检查：resolved == rootReal 或 hasPrefix(resolved, rootReal+sep)，否则 ErrPathTraversal
```
要点：前缀检查在符号链接解析之后；`rootReal` 启动时解析一次。

## 11. 测试点

- **util.SafeResolve**：正常路径、`..` 穿越、绝对路径、symlink 逃逸、不存在目标（父目录解析）、根本身
- **util.token**：生成长度、哈希稳定性
- **config**：缺 `--root` 失败、参数 > 环境变量优先级、TTL 解析
- **store**：建用户唯一约束、会话增删查、过期会话被清理
- **service.auth**：首启建管理员、登录成功/失败、连续失败锁定、登出删会话、改密吊销其他会话、弱密码拒绝
- **middleware.auth**：无 token / 错 token / 过期 token → 401，有效 token 注入用户
- **handler e2e**（httptest）：login → 带令牌 me → logout → 再 me 401
- **web 静态**：未知子路由回退 index.html；`/api/x` 不被静态接管
